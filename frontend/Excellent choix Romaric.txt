Panier (Cart/Basket)

    Table Panier avec les livres ajoutés avant finalisation
    Relation User ↔ Panier (One-to-One) : Un utilisateur a un panier actif
    Relation Panier ↔ LignePanier (One-to-Many) : Un panier contient plusieurs lignes
    Relation LignePanier ↔ Livre (Many-to-One) : Chaque ligne référence un livre

Commandes (Orders)

    Table Commande pour remplacer/compléter le modèle Achat actuel
    Relation User ↔ Commande (One-to-Many) : Un utilisateur peut avoir plusieurs commandes
    Relation Commande ↔ LigneCommande (One-to-Many) : Une commande contient plusieurs lignes
    Relation LigneCommande ↔ Livre (Many-to-One) : Chaque ligne référence un livre

Flux recommandé :

    Panier → Ajouter/retirer des livres
    Commande → Finaliser le panier (checkout)
    Achat → Confirmation de paiement et traitement
Créer un code pour gérer les paniers, les commandes et les achats sur un site de vente et de prêt de livres nécessite plusieurs composants. Voici une explication de la structure et des concepts que vous utiliseriez, en vous basant sur un cadre de développement web comme Python avec Django ou Node.js avec Express, qui sont couramment utilisés pour ce genre de projet.

-----

### 1\. La Base de Données

Tout commence par la base de données. Vous devez concevoir des modèles pour stocker les informations de manière organisée. Voici les entités principales dont vous auriez besoin :

  * **`Utilisateur`** : Contient les informations de chaque membre (nom, adresse e-mail, mot de passe, etc.).
  * **`Livre`** : Contient les détails de chaque livre disponible (titre, auteur, ISBN, prix de vente, prix de prêt, disponibilité).
  * **`Panier`** : C'est le lien entre un **`Utilisateur`** et les **`Livre`** qu'il souhaite acheter ou emprunter. Ce n'est pas une simple liste. Il s'agit généralement d'un modèle qui peut contenir plusieurs **`ArticlesPanier`**.
  * **`ArticlePanier`** : Représente une ligne dans le panier. Il contient une référence au **`Panier`**, au **`Livre`** et spécifie si l'action est un **achat** ou un **emprunt**.
  * **`Commande`** : Une fois le panier validé, il devient une **`Commande`**. Ce modèle enregistre l'état de la transaction (en cours, payée, expédiée), la date, le montant total et une référence à l'**`Utilisateur`**.
  * **`ArticleCommande`** : Similaire à l'**`ArticlePanier`**, mais pour les commandes. Il contient une référence à la **`Commande`**, au **`Livre`** et à l'action (achat ou emprunt).

Voici un exemple simplifié en pseudo-code pour illustrer la structure de ces modèles :

```python
# Modèle en pseudo-code (inspiré de Django ORM)

class Utilisateur:
    id = PrimaryKey()
    nom = CharField()
    email = EmailField()
    # ... autres champs

class Livre:
    id = PrimaryKey()
    titre = CharField()
    auteur = CharField()
    prix_achat = DecimalField()
    prix_pret = DecimalField()
    disponible_achat = BooleanField(default=True)
    disponible_pret = BooleanField(default=True)

class Panier:
    id = PrimaryKey()
    utilisateur = ForeignKey(Utilisateur)
    cree_le = DateTimeField(auto_now_add=True)

class ArticlePanier:
    id = PrimaryKey()
    panier = ForeignKey(Panier)
    livre = ForeignKey(Livre)
    type_action = CharField(choices=[('ACHAT', 'Achat'), ('PRET', 'Prêt')])
    quantite = IntegerField(default=1)

class Commande:
    id = PrimaryKey()
    utilisateur = ForeignKey(Utilisateur)
    date_commande = DateTimeField(auto_now_add=True)
    total = DecimalField()
    statut = CharField(choices=[('EN_COURS', 'En cours'), ('PAYEE', 'Payée'), ('EXPEDIEE', 'Expédiée')])

class ArticleCommande:
    id = PrimaryKey()
    commande = ForeignKey(Commande)
    livre = ForeignKey(Livre)
    type_action = CharField()
    prix_unitaire = DecimalField()
```

### 2\. La Logique du Panier (Frontend et Backend)

La gestion du panier se fait en deux parties : une partie côté client (**frontend**) et une partie côté serveur (**backend**).

#### Côté Frontend (JavaScript)

Quand un utilisateur clique sur "Ajouter au panier", le navigateur envoie une requête au serveur. Il est courant d'utiliser une requête **AJAX** (ou `fetch`) pour éviter de recharger la page. Cette requête envoie les informations essentielles : l'ID du livre et l'action (achat ou prêt).

```javascript
// Exemple de code JavaScript
document.getElementById('bouton-ajouter').addEventListener('click', function() {
    const livreId = this.dataset.livreId;
    const action = this.dataset.action; // 'achat' ou 'pret'

    fetch('/api/panier/ajouter', {
        method: 'POST',
        headers: {
            'Content-Type': 'application/json',
        },
        body: JSON.stringify({ livreId: livreId, action: action })
    })
    .then(response => response.json())
    .then(data => {
        // Mettre à jour l'affichage du panier sur la page
        alert('Livre ajouté au panier !');
    })
    .catch(error => {
        console.error('Erreur:', error);
    });
});
```

#### Côté Backend (Python/Node.js)

Le serveur reçoit la requête. Voici la logique qui se met en place :

1.  **Identifier l'utilisateur** : Le serveur doit savoir qui est connecté. Cela se fait généralement via une **session** ou un **token** d'authentification.
2.  **Trouver le panier de l'utilisateur** : Il vérifie si l'utilisateur a déjà un panier actif. S'il n'en a pas, il en crée un nouveau.
3.  **Vérifier la disponibilité** : Le code vérifie si le livre est bien disponible pour l'action demandée (achat ou prêt).
4.  **Ajouter l'article au panier** :
      * Si un article pour ce livre existe déjà dans le panier, il met à jour la quantité.
      * Sinon, il crée un nouvel `ArticlePanier` dans la base de données.
5.  **Envoyer une réponse** : Le serveur renvoie une réponse JSON indiquant le succès de l'opération.

-----

### 3\. Le Processus de Commande et de Paiement

Une fois que l'utilisateur est satisfait de son panier, il passe à la caisse.

#### Côté Backend (Logique de Commande)

Le code côté serveur gère ce processus en plusieurs étapes :

1.  **Validation du panier** : Le serveur recalcule le total et vérifie à nouveau la disponibilité des livres au cas où un autre utilisateur les aurait achetés entre-temps.
2.  **Création de la `Commande`** : Un nouvel objet `Commande` est créé dans la base de données, avec le statut "en cours".
3.  **Transfert des articles** : Les `ArticlePanier` sont copiés pour devenir des `ArticleCommande`. Le panier de l'utilisateur est ensuite vidé.
4.  **Intégration d'un Service de Paiement** : Pour les achats, le code interagit avec une API de paiement externe comme **Stripe** ou **PayPal**.  Le serveur envoie les détails de la commande (montant, devises) à l'API de paiement, qui renvoie une URL ou un token pour le paiement sécurisé.
5.  **Confirmation de la transaction** : Après le paiement, le service externe envoie une notification (via un **webhook**) à votre serveur pour confirmer que la transaction a réussi. C'est à ce moment-là que vous mettez à jour le statut de la `Commande` en "payée" et que vous mettez à jour la disponibilité des livres.

-----

### 4\. La Gestion des Emprunts

Les emprunts suivent une logique similaire, mais au lieu de déclencher un paiement, la validation de la commande pourrait :

  * Déclencher une transaction pour un petit coût de prêt.
  * **Créer un enregistrement de prêt** : Un nouveau modèle pourrait être créé pour suivre l'état du prêt (date de début, date de retour prévue, statut "en cours", "en retard", etc.).
  * **Notifier l'utilisateur** : Envoyer un e-mail de confirmation.

La gestion de ces processus se fait en utilisant des **endpoints d'API** dédiés (par exemple, `/api/panier/ajouter`, `/api/commande/creer`, `/api/paiement/valider`) qui gèrent la logique métier décrite ci-dessus.

# Comment appeler ton API Django depuis un frontend React

---

## 1. Principe

* Ton backend Django expose une API REST (par exemple via Django REST Framework).
* Ton frontend React doit envoyer des requêtes HTTP (GET, POST, PUT, DELETE) à ces endpoints.
* Les données échangées sont généralement en JSON.
* React reçoit la réponse, la traite, et affiche ou utilise les données.

---

## 2. Méthode principale : fetch (natif JS) ou axios (librairie)

### Exemple simple avec **fetch** :

```js
// GET liste de livres
fetch('http://localhost:8000/api/livres/')
  .then(response => {
    if (!response.ok) throw new Error('Erreur réseau');
    return response.json();
  })
  .then(data => {
    console.log('Données reçues:', data);
    // Ici tu peux mettre à jour ton state React
  })
  .catch(error => {
    console.error('Erreur:', error);
  });
```

### Exemple avec **axios** (plus puissant et pratique) :

```bash
npm install axios
```

```js
import axios from 'axios';

axios.get('http://localhost:8000/api/livres/')
  .then(response => {
    console.log(response.data);
  })
  .catch(error => {
    console.error('Erreur:', error);
  });
```

## 5. Utiliser React state

Exemple rapide avec hooks :

```js
import React, { useEffect, useState } from 'react';

function Livres() {
  const [livres, setLivres] = useState([]);

  useEffect(() => {
    fetch('http://localhost:8000/api/livres/')
      .then(res => res.json())
      .then(data => setLivres(data))
      .catch(console.error);
  }, []);

  return (
    <div>
      <h1>Livres</h1>
      <ul>
        {livres.map(livre => (
          <li key={livre.id}>{livre.titre} par {livre.auteur}</li>
        ))}
      </ul>
    </div>
  );
}

export default Livres;
```

import { useState } from "react";
import axios from "axios";

export function AddLivreForm() {
  const [formData, setFormData] = useState({
    titre: "",
    auteur: "",
    description: "",
    fichier_pdf: null,
  });

  function handleChange(e) {
    const { name, value, files } = e.target;
    setFormData({
      ...formData,
      [name]: files ? files[0] : value,
    });
  }

  async function handleSubmit(e) {
    e.preventDefault();
    const data = new FormData();
    data.append("titre", formData.titre);
    data.append("auteur", formData.auteur);
    data.append("description", formData.description);
    if (formData.fichier_pdf) {
      data.append("fichier_pdf", formData.fichier_pdf);
    }

    try {
      const res = await axios.post(
        "http://localhost:8000/api/livres/add/",
        data,
        { headers: { "Content-Type": "multipart/form-data" } }
      );
      alert("Livre ajouté avec succès !");
      console.log(res.data);
    } catch (err) {
      console.error(err);
      alert("Erreur lors de l'ajout du livre");
    }
  }

  return (
    <form onSubmit={handleSubmit} className="flex flex-col gap-2 w-1/2">
      <input
        type="text"
        name="titre"
        placeholder="Titre"
        value={formData.titre}
        onChange={handleChange}
        required
      />
      <input
        type="text"
        name="auteur"
        placeholder="Auteur"
        value={formData.auteur}
        onChange={handleChange}
        required
      />
      <textarea
        name="description"
        placeholder="Description"
        value={formData.description}
        onChange={handleChange}
      ></textarea>
      <input
        type="file"
        name="fichier_pdf"
        accept="application/pdf"
        onChange={handleChange}
      />
      <button type="submit" className="bg-green-700 text-white p-2">
        Ajouter le livre
      </button>
    </form>
  );
}

### **`ListeLivres.jsx`** avec **debounce**

```jsx
import { useState, useEffect } from "react";
import FiltreLivres from "./FiltreLivres";

export default function ListeLivres() {
  const [livres, setLivres] = useState([]);
  const [filtres, setFiltres] = useState({ categorie: "", nom: "" });
  const [debouncedFiltres, setDebouncedFiltres] = useState(filtres);

  // Debounce : attend 300ms avant de mettre à jour debouncedFiltres
  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedFiltres(filtres);
    }, 300);

    return () => clearTimeout(handler); // annule si l'utilisateur tape encore
  }, [filtres]);

  // Appel API quand debouncedFiltres change
  useEffect(() => {
    let url = "http://localhost:8000/api/livres/";
    const params = [];

    if (debouncedFiltres.categorie)
      params.push(`categorie=${debouncedFiltres.categorie}`);
    if (debouncedFiltres.nom)
      params.push(`nom=${debouncedFiltres.nom}`);

    if (params.length > 0) url += `?${params.join("&")}`;

    fetch(url)
      .then((res) => res.json())
      .then((data) => setLivres(data))
      .catch((err) => console.error("Erreur API :", err));
  }, [debouncedFiltres]);

  return (
    <div>
      <FiltreLivres onFiltreChange={setFiltres} />

      <ul>
        {livres.map((livre) => (
          <li key={livre.id}>
            {livre.titre} ({livre.categorie})
          </li>
        ))}
      </ul>
    </div>
  );
}
```

### Pourquoi c’est mieux ?

* Sans debounce → 10 lettres = 10 requêtes API
* Avec debounce → 10 lettres tapées rapidement = 1 seule requête après 300ms d’inactivité
* Ça évite de surcharger ton backend Django

from .models import Livre

from .pagination import SmallResultsSetPagination

class LivreListView(generics.ListAPIView):
    serializer_class = LivreSerializer
    permission_classes = [AllowAny]
    pagination_class = SmallResultsSetPagination

    def get_queryset(self):
        qs = Livre.objects.select_related('categorie').all()
        # filtres
        categorie = self.request.GET.get('categorie')
        nom = self.request.GET.get('nom')
        langue = self.request.GET.get('langue')

        if categorie:
            # accepte soit id soit nom (string)
            if categorie.isdigit():
                qs = qs.filter(categorie_id=int(categorie))
            else:
                qs = qs.filter(categorie__name__icontains=categorie)
        if nom:
            qs = qs.filter(titre__icontains=nom)  # recherche partielle
        if langue:
            qs = qs.filter(langue__iexact=langue)

        return qs.order_by('titre')

### b) hook custom `useLivres` (`src/hooks/useLivres.js`)

```javascript
import { useState, useEffect, useCallback } from "react";
import { fetchLivres } from "../api/livres";

export function useLivres(initialFilters = { nom: "", categorie: "" }, initialPage=1) {
  const [filtres, setFiltres] = useState(initialFilters);
  const [debouncedFiltres, setDebouncedFiltres] = useState(initialFilters);
  const [page, setPage] = useState(initialPage);
  const [pageSize, setPageSize] = useState(12);
  const [data, setData] = useState({ results: [], count: 0, next: null, previous: null });
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);

  // debounce 300ms
  useEffect(() => {
    const id = setTimeout(() => setDebouncedFiltres(filtres), 300);
    return () => clearTimeout(id);
  }, [filtres]);

  const load = useCallback(async (debouncedFiltres, page, pageSize) => {
    setLoading(true);
    setError(null);
    try {
      const params = {
        nom: debouncedFiltres.nom || undefined,
        categorie: debouncedFiltres.categorie || undefined,
        page,
        page_size: pageSize,
      };
      const result = await fetchLivres(params);
      setData(result); // attendu: {count, next, previous, results}
    } catch (err) {
      setError(err.message || "Erreur réseau");
    } finally {
      setLoading(false);
    }
  }, []);

  // reload quand debouncedFiltres, page ou pageSize changent
  useEffect(() => {
    load(debouncedFiltres, page, pageSize);
  }, [debouncedFiltres, page, pageSize, load]);

  return {
    livres: data.results,
    total: data.count,
    next: data.next,
    previous: data.previous,
    loading,
    error,
    filtres,
    setFiltres,
    page,
    setPage,
    pageSize,
    setPageSize,
    reload: () => load(debouncedFiltres, page, pageSize),
  };
}
```

## 3) Création d’un filtre avancé (`filters.py`)

# app/filters.py
import django_filters
from .models import Livre

class LivreFilter(django_filters.FilterSet):
    # recherche partielle sur titre
    titre = django_filters.CharFilter(field_name='titre', lookup_expr='icontains')
    # recherche partielle sur nom de la catégorie
    categorie = django_filters.CharFilter(field_name='categorie__name', lookup_expr='icontains')
    # recherche exacte sur id catégorie
    categorie_id = django_filters.NumberFilter(field_name='categorie_id')
    # filtre exact sur langue
    langue = django_filters.CharFilter(field_name='langue', lookup_expr='iexact')
    # plage de dates
    date_publication_min = django_filters.DateFilter(field_name='date_publication', lookup_expr='gte')
    date_publication_max = django_filters.DateFilter(field_name='date_publication', lookup_expr='lte')
    # plage de prix
    prix_min = django_filters.NumberFilter(field_name='prix_achat', lookup_expr='gte')
    prix_max = django_filters.NumberFilter(field_name='prix_achat', lookup_expr='lte')

    class Meta:
        model = Livre
        fields = [
            'titre',
            'categorie',
            'categorie_id',
            'langue',
            'date_publication_min',
            'date_publication_max',
